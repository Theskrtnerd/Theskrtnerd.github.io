11:T1876,<h2>Introduction</h2>
<p>This semester, I had the chance to study the course <a href="https://www.adelaide.edu.au/course-outlines/001956/2/sem-1/">Computer Systems</a>. This course is (kinda) based on the course <a href="https://www.nand2tetris.org">nand2tetris</a> and the book <a href="https://www.amazon.com/Elements-Computing-Systems-Building-Principles/dp/0262640686/">The Elements of Computing Systems</a>. The course is famous and notorious for being hard, but it teaches you all the fundamentals of Computing Systems, from high-level languages like Python and C++, all the way down to machine code with 0 and 1 bits. </p>
<p>So it might be obvious that this course is regarded as the hardest in my entire CS degree. Furthermore, you can probably guess how hard the assignments were. Especially in the recent assignment, we were tasked with writing <a href="https://www.jk-quantized.com/experiments/HomebrewComputer/Cheatsheets/hackASM.html">Hack Assembly</a> code, which is an Assembly language that is created for the Hack Platform, with all basic components of a normal computer with CPU and RAM, etc. This language at first looks super hard to read, with some weird syntax. Below is a small snippet of what the code looks like.</p>
<pre><code class="hljs language-sample.asm">@R1
D=M
@x
M=D // x = R1

@R2
D=M
@y
M=D // y = R2

@0
D=A
@R3
M=D // R3 = 0
</code></pre>
<h2>Struggling with Sorting</h2>
<p>I struggled with this assignment a lot. But of all the tasks, the task that interested me the most was the task of writing a sorting algorithm in a file called <code>ArrSort.asm</code> using this language.</p>
<p>This task was not that difficult, but what was challenging was the bonus part, where you can receive bonus marks for implementing an algorithm in <code>O(nlog(n))</code> time complexity. My friend Khanh even talked about not being able to solve this before I started this assignment.</p>
<p>At first, I thought this was going to be a simple task. But then I was (obviously) wrong. The two most famous sorting algorithms that have a <code>O(nlog(n))</code> time complexity are <strong>Merge Sort</strong> and <strong>Quick Sort</strong>.</p>
<p>I first started by trying to implement <strong>Quick Sort</strong> in Assembly. But after trying to implement it for 3 hours, I realized a major issue with this algorithm: This algorithm needs to be implemented recursively, while Hack Assembly cannot implement recursion because this language doesn't have a return address after jumping and PC (Program Counter) integrated. This was a huge shock and letdown for me, as I've spent 3 hours (late night) for nothing.</p>
<p>After that, I changed to <strong>Merge Sort</strong> and this was a much better solution that can be implemented iteratively. With help from ChatGPT, I was able to come up with this Python code for a high-level implementation:</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">def</span> <span class="hljs-title function_">merge</span>(<span class="hljs-params">arr, r4, r5, r6</span>):
    i = r4
    j = r5 + <span class="hljs-number">1</span>
    <span class="hljs-keyword">while</span> i &#x3C;= r5 <span class="hljs-keyword">and</span> j &#x3C;= r6:
        <span class="hljs-keyword">if</span> arr[i] &#x3C;= arr[j]:
            i += <span class="hljs-number">1</span>
        <span class="hljs-keyword">else</span>:
            temp = arr[j]
            <span class="hljs-keyword">for</span> k <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(j, i, -<span class="hljs-number">1</span>):
                arr[k] = arr[k - <span class="hljs-number">1</span>]
            arr[i] = temp
            i += <span class="hljs-number">1</span>
            j += <span class="hljs-number">1</span>
            r5 += <span class="hljs-number">1</span>
    
<span class="hljs-keyword">def</span> <span class="hljs-title function_">merge_sort</span>(<span class="hljs-params">arr</span>):
    r2 = <span class="hljs-built_in">len</span>(arr)
    r3 = <span class="hljs-number">1</span>
    <span class="hljs-keyword">while</span> r3 &#x3C; r2:
        r4 = <span class="hljs-number">0</span>
        <span class="hljs-keyword">while</span> r4 &#x3C; r2 - <span class="hljs-number">1</span>:
            r7 = r2 - <span class="hljs-number">1</span>
            r5 = r3+r4-<span class="hljs-number">1</span>
            r5 = <span class="hljs-built_in">min</span>(r5, r7)
            r6 = r5+r3
            r6 = <span class="hljs-built_in">min</span>(r6, r7)
            merge(arr, r4, r5, r6)
            r4 += <span class="hljs-number">2</span> * r3
        r3 *= <span class="hljs-number">2</span>
</code></pre>
<p>Notice how the variables are named from <code>r2</code> to <code>r7</code>. This is because in the Hack Assembly, the variables you use will mostly be from <code>R1</code> to <code>R15</code>. With this naming, it will be easier for me to convert all this code to Hack Assembly. And after another 2 hours of work, I finally came up with this super-long code:</p>
<pre><code class="hljs language-asm">@3
M=1 // r3=1
(WHILEA)
@3
D=M
@2
D=D-M
@END
D;JGE // while r3&#x3C;r2
@4
M=0 // r4 = 0
(WHILEB)
@4
D=M+1
@2
D=D-M
@ENDWHILEB
D;JGE // while r4 &#x3C; r2-1
@2
D=M
@7
M=D-1 // r7 = r2-1
@3
...
</code></pre>

<p>So in conclusion, I've spent six hours (an all-nighter) just for one extra mark. But at least for me, it was super worth it, as I was able to understand assembly code deep down to the RAM and ROM level. Furthermore, I now understand more about sorting algorithms and how they work.</p>
<p>What's even funnier is that I neglected all testing, which contributes many more marks than this bonus part. And luckily, with the automatic extension, I was able to submit all the testing files just one day before the final deadline.</p>
